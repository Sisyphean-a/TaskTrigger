# Android 定时指令执行工具 - 产品需求文档

## 项目名称

**TaskTrigger**

一个轻量级 Android 定时任务执行工具。

目标：

> 用户设置一个时间，到时间后自动执行一条系统指令。

例如：

- 定时启动飞书
- 定时打开某个 App
- 定时执行 Shell 命令
- 定时执行 Root 指令
- 定时运行维护脚本

核心理念：

**添加任务 → 保存 → 退出 App → 系统负责唤醒执行**

应用本身不常驻后台，不使用悬浮窗，不使用无障碍服务。

---

# 核心使用场景

## 场景 1：自动启动飞书

用户配置：

```
时间：
工作日 18:00

命令：

monkey -p com.ss.android.lark \
-c android.intent.category.LAUNCHER 1
```

执行效果：

每天 18:00 自动启动飞书。

---

## 场景 2：Root 用户执行系统命令

例如：

```
am force-stop com.xxx.app

am start xxx
```

或者：

```
pm disable-user xxx
```

执行方式：

```
su -c "command"
```

---

# 设计原则

## 1. 不常驻后台

禁止：

- Foreground Service
- 常驻通知
- 悬浮按钮
- 无障碍服务

应用平时应该处于：

```
Stopped
 |
 |
AlarmManager
 |
 |
时间到
 |
 |
BroadcastReceiver
 |
 |
执行任务
```

---

## 2. 极低资源占用

正常状态：

- CPU：0%
- 内存：接近 0
- 无网络请求
- 无后台轮询

↓

# 第一版本目标（MVP）

只实现：

1. 添加任务

2. 设置时间

3. 保存任务

4. 到时间执行 Shell

5. Root 执行支持

6. 查看执行日志


完成这个版本，就已经满足 90% 使用需求。
